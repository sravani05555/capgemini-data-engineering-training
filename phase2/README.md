# 🚀 Phase 2 – SQL to PySpark (Joins & Aggregations)

## 📌 Objective
In this phase, I focused on applying SQL concepts like joins and aggregations using PySpark on real-world datasets

## 💡 My Approach
First, I solved all problems using SQL and then converted the logic into PySpark. This helped me understand how joins and aggregations work in both approaches.

## ✅ Key Learnings
- Using join() for combining datasets
- Apply groupBy() with sum() and avg()
- filter() for conditions after aggregation
- orderBy() for sorting results

## 🧠 What I Understood
I understood how to combine multiple tables using joins and how aggregations help in getting meaningful insights like total values and averages.

## ⚠️ Challenges Faced
- Writing correct join conditions 
- Understanding the difference between left join and inner join
- Applying groupBy with multiple operations

## 🔧 How I Solved Them
- Practiced SQL and PySpark side-by-side
- Broke problems into smaller steps
- Verified outputs after each step

## 🛠 Tools Used
- DB-Fiddle for SQL
- Spark Playground for PySpark

## 📊 Outcome
I am now able to perform joins and aggregations in PySpark and understand how SQL logic maps to PySpark transformations.
